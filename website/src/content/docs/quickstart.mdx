## Prerequisites

<p>All builds need: a working <code>git</code>, a Rust toolchain (<code>rustup</code>), and the platform's native build tools listed below. The repo ships a <code>script/bootstrap</code> entrypoint that installs the right set for your OS.</p>

## 1. Clone

<CodeCard
  file="terminal"
  lang="bash"
  html={`git clone https://github.com/mehmetbaykar/zap.git zap
<span class="key">cd</span> zap`}
/>

## 2. Install build dependencies

<p>Pick your platform. The bootstrap script delegates to <code>script/macos/bootstrap</code>, <code>script/linux/bootstrap</code>, or <code>script/windows/bootstrap.ps1</code> automatically.</p>

## 3. Build & run

<p>Zap ships two binaries: <code>zap-oss</code> (no cloud-bound features compiled in — the right target for community builds) and <code>warp</code> (Warp Inc.'s internal channel, requires their private channel config). Use <code>zap-oss</code>.</p>

<TabsCard tabs={[
  {
    key: 'run',
    label: 'script/run (recommended)',
    html: `<span class="com"># Cross-platform: detects OSS channel automatically, builds, launches</span>
./script/run

<span class="com"># Pass --release for an optimized build</span>
./script/run --release`
  },
  {
    key: 'cargo',
    label: 'cargo',
    html: `<span class="com"># Direct cargo invocation — useful for IDE integration / debugging</span>
cargo run --release <span class="acc">--bin zap-oss</span> --features gui

<span class="com"># Dev profile (faster compile, slower runtime)</span>
cargo run <span class="acc">--bin zap-oss</span> --features gui`
  },
  {
    key: 'bundle',
    label: 'Bundle a release',
    html: `<span class="com"># Produces a platform installer in target/release/bundle/</span>
<span class="com">#   macOS  → .app + .dmg via cargo-bundle / create-dmg</span>
<span class="com">#   Linux  → AppImage via linuxdeploy</span>
<span class="com">#   Windows→ .exe installer via Inno Setup</span>
./script/bundle --release`
  }
]} />

## 4. Configure a provider

<p>After launch, open <code>Settings → Agents → Providers</code> and add at least one OpenAI-compatible endpoint. Credentials are written to <code>settings.toml</code> on disk and never sent anywhere except the endpoint you configure.</p>

<CodeCard
  file="~/.config/zap/settings.toml"
  lang="TOML"
  html={`<span class="key">[providers.deepseek]</span>
base_url = <span class="str">"https://api.deepseek.com/v1"</span>
protocol = <span class="str">"deepseek"</span>     <span class="com"># or: openai, anthropic, gemini, ollama</span>
api_key  = <span class="str">"sk-•••"</span>          <span class="com"># stored locally, never sent home</span>

<span class="key">[[providers.deepseek.models]]</span>
display = <span class="str">"DeepSeek V4 Pro"</span>
id      = <span class="str">"deepseek-v4-pro"</span>
context = <span class="num">1000000</span>
output  = <span class="num">384000</span>`}
/>

<p class="muted">Hit <code>⌘P</code> to switch model, <code>/</code> to open the slash menu. Cloud Agent, Computer Use, and telemetry ship disabled — see the GitHub repo for the full settings reference.</p>
